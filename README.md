# SoundCloud Menu Extension

Tampermonkeyユーザースクリプト。SoundCloud上でジャケット画像をクリップボードにコピーする機能と、ダウンロードしたファイルにタイトル・アーティスト・アルバム・ジャンル・ジャケット画像のタグを自動で埋め込む機能を追加する。

## 機能

### 1. ヘッダーのジャケットコピーボタン

トラックページのヘッダー、検索ボックスの右にボタンを追加する。クリックすると、今開いているトラックのジャケット画像（可能な限り高解像度）をクリップボードにコピーする。

### 2. 一覧ページのジャケットコピーボタン

`https://soundcloud.com/you/likes`などのトラック一覧ページでは、各トラックの既存のアクションボタン列（Like/Follow/Moreや、Like/Repost/Share/Copy Linkなど）に並べてコピーボタンを設置する。グリッド表示（Badges）・リスト表示（List）の両方に対応する。トラック単体ページでも、波形下のLike/Repost/Share/Copy Link/Moreの並びに同様のボタンを設置する。

### 3. Download file with metadata

トラックの「More」メニューを開くと、既存の「Download file」の下に「Download file with metadata」が追加される。クリックすると、ファイルをダウンロードする際にタイトル・アルバム名（トラックのタイトルと同一）・アーティスト名・ジャンル・ジャケット画像を書き込んでから保存する。**既にタグが設定されている項目は上書きせず、未設定の項目だけを埋める。**

対応形式:
- **WAV** — `LIST/INFO`チャンク（`INAM`=タイトル、`IART`=アーティスト、`IPRD`=アルバム、`IGNR`=ジャンル）に加えて、非標準だがffmpeg/mutagenなどが採用している`id3 `チャンク（中身は丸ごとID3v2タグ、`APIC`フレームでジャケット画像を保持）にも書き込む。WAV自体の仕組みにはジャケットを埋め込む標準的な方法が無いための対応。自前実装（`id3 `チャンクの構築にはMP3と共通のID3v2タグ生成ロジックを再利用）。
- **MP3** — ID3v2タグ（`TIT2`/`TALB`/`TPE1`/`TCON`/`APIC`）に書き込む。既存タグの読み取りは自前実装、書き込みは外部ライブラリ`browser-id3-writer`（バージョン固定・ソース確認済み、CDNから実行時に動的import）を使用する。
- **FLAC** — `VORBIS_COMMENT`ブロック（`TITLE`/`ARTIST`/`ALBUM`/`GENRE`）と`PICTURE`ブロックに書き込む。自前実装（外部ライブラリ不要）。
- 上記以外の形式（m4aなど）は無加工でダウンロードされる。

いずれの機能もクリック時にボタンが読み込み中→成功（緑）/失敗（赤）のアイコンに変化する。失敗時は画面右上にトースト通知で理由も表示される（ブラウザの言語設定が日本語なら日本語、それ以外は英語）。

## インストール

1. Tampermonkeyをインストールする。
2. Tampermonkeyのダッシュボードで「新規スクリプトを作成」を選び、`soundcloud-artwork-copier.user.js`の内容を貼り付けて保存する。

## 動作条件・制限

- ログイン済みのSoundCloudを対象とする。
- ヘッダーのコピーボタンはトラック単体ページ（`og:type` が `music.song`）でのみ機能する。プレイリストやプロフィールページではボタンを押すと失敗表示になる。
- 一覧ページのコピーボタンは、サムネイル画像がまだ遅延読み込みされていない場合は失敗表示になる。
- 「Download file with metadata」は、SoundCloudの非公開API（`api-v2.soundcloud.com`の`/resolve`と`/tracks/{id}/download`）を利用している。ページHTMLを直接`fetch`する方式は、SoundCloudのbot対策（DataDome）によりモバイル版へリダイレクトされCORSでブロックされるため使用していない。認証には`oauth_token`クッキーの値を`Authorization`ヘッダーとして付与している。
- MP3のID3v2タグ書き込みでは、タイトル/アルバム/アーティスト/ジャンル/ジャケット以外の既存タグ（例: コメント）は保持されない（`browser-id3-writer`が既存タグを丸ごと消して新規に書き直す仕様のため）。WAV・FLACは対象外のチャンク/ブロックはそのまま保持される。
